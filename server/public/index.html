Here's a description of routes

/api GET routes:

/stats — return count of queries from mobile and non-mobile devices

/bookings/total — number of bookings in database

/bookings/next_days?days=n — number of bookings for each day separately in all next n days. if days not specified defaults to 7

/bookings/page/p — most recent bookings. 15 per page. If p not specified default to 1

/bookings/i — return booking with id = i

/tables/free?date_booked_for — return free tables at specified date and time

/income — total income of restaurant from all orders

/menu/i — return menu with id = i. Default is 1, 'coz we have only one menu in db

/menu/item/ — return one particular menu item with specified id from specified menu (in url queries)


/authenticate POST — return access token if username and password are correct
